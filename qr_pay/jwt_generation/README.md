# JWT Generation (PayNet DuitNow Pay)

This package implements **JSON Web Token (JWT)** for PayNet DuitNow Pay API authentication, as per [PayNet DuitNow Pay – API Authentication](https://docs.developer.paynet.my/docs/duitnow-pay/integration/api-authentication).

## Overview

- **Generate JWT** for outbound API requests and set `Authorization: Bearer <token>`.
- **Verify JWT** on API responses using PayNet’s public key (certificate). Verification checks signature and expiration only; there is no payload hash (`ds`) in the Pay scheme.

Algorithm: **RS256**.

## JWT claims (per Pay doc)

| Claim | Value |
|-------|--------|
| **exp** | Expiry date & time (within 1 hour). |
| **iss** | BIC code assigned during onboarding. |
| **sub** | Merchant ID assigned during merchant registration. |
| **jti** | UUID v4 generated by merchant. |
| **key** | Project ID assigned during onboarding. |
| **data** | Request payload in body format (the same body you send in the API request). |

## Usage

### 1. Generate JWT for an API request

```go
import "example.com/sample-repo/qr_pay/jwt_generation"

// Load private key (default: sample_private_key.key in this package folder).
privateKey, err := jwt_generation.LoadDefaultPrivateKey()
// Or: privateKey, err := jwt_generation.LoadPrivateKey("path/to/private.key")
if err != nil {
    return err
}

// Request body: same object you will send in the API request (becomes the "data" claim).
requestData := map[string]interface{}{
    "checkoutId": checkoutID,
    "amount":     "10.00",
    "merchant":   map[string]interface{}{"productId": "P00000205"},
    // ... rest of request body
}

token, err := jwt_generation.GenerateJWT(jwt_generation.GenerateOptions{
    PrivateKey: privateKey,
    Algorithm:  jwt_generation.RS256,
    Issuer:     "<acquirerId>",   // BIC code
    Subject:    "<merchantId>",   // Merchant ID
    JTI:        checkoutID,       // UUID v4
    Key:        "<api-key-name>", // Project ID
    Data:       requestData,
})
if err != nil {
    return err
}

req.Header.Set("Authorization", "Bearer "+token)
```

### 2. Verify JWT on an API response

```go
publicKey, _, err := jwt_generation.LoadDefaultCertificate()
// Or: publicKey, _, err := jwt_generation.LoadCertificate("path/to/paynet-cert.cer")
if err != nil {
    return err
}

token := r.Header.Get("Authorization")

err = jwt_generation.VerifyJWT(jwt_generation.VerifyOptions{
    Token:     token,
    PublicKey: publicKey,
    Algorithm: jwt_generation.RS256,
})
if err != nil {
    return err // invalid or expired token
}
```

Verification validates the signature and `exp` only. PayNet Pay response JWTs do not use a `ds` (payload hash) claim.

### 3. Key and certificate files

Place your key and certificate in the `jwt_generation` folder with these names to use the default loaders:

- **sample_private_key.key** – PEM, either `RSA PRIVATE KEY` (PKCS#1) or `PRIVATE KEY` (PKCS#8).
- **sample_external_certificate.cer** – PEM with PayNet’s X.509 certificate (for verification).

For custom paths, use `LoadPrivateKey(path)` and `LoadCertificate(path)`.

To generate a key pair (e.g. for sandbox):

```bash
openssl req -newkey rsa:2048 -nodes -keyout example.key -out example.csr
openssl x509 -signkey example.key -in example.csr -req -days 365 -out example.cer
```

## Specification summary

| Step | Description |
|------|-------------|
| 1 | JWT **header**: `{"alg":"RS256","typ":"JWT"}`. |
| 2 | JWT **claims**: `exp`, `iss`, `sub`, `jti`, `key`, `data` (request body object). |
| 3 | Sign `base64url(header).base64url(payload)` with the private key (RS256). |
| 4 | Token = `header.payload.signature`; send as `Authorization: Bearer <token>`. |

Expiry (`exp`) must be within 1 hour of issue per PayNet. Default in this package is 1 hour.
